### Context

**需求背景**: 现有平台需要实现一个报表中心子模块，核心功能包括自定义报表模板管理、报表创建管理和报表发布管理，以满足报表全流程自定义需求。用户是 Java 后端开发，前端是 Vue 项目，不再使用 FineReport。

**目标**: 建立一个灵活、可扩展的报表解决方案，允许用户自定义报表样式、数据源和展现方式，并能方便地进行管理和发布。核心理念是利用 SQL 作为数据定义语言，前端负责交互式渲染，后端提供数据接口和文件导出能力。

### 总体技术架构设计 (SQL + 前端渲染 + 后端导出)

我们将采用前后端分离的架构，并充分利用现有技术栈 (Java, Vue)，引入数据可视化库。

```
+-----------------------------------+
|          前端 (Vue.js)            |
|                                   |
| - 报表设计器 (可视化配置生成)       |<------------------+
| - 模板管理界面                      |                   |
| - 报表预览/管理 (报表渲染器)        |<------------------+
| - 数据源管理界面                    |                   |
+-----------------------------------+                   |
           | RESTful API                               |
           v                                           |
+---------------------------------------------------+    |
|                  后端 (Java)                      |    |
|                                                   |    |
| - API 网关/服务 (Auth/Rate Limit)                  |    |
| - 报表模板管理服务 (CRUD 报表定义元数据)         |<----+
| - 报表实例管理服务 (CRUD 报表实例配置)           |<----+
| - SQL 查询执行服务 (安全执行 SQL, 返回原始数据)  |
| - 数据源管理服务 (CRUD 数据源配置, 加密敏感信息) |
| - 报表发布服务 (控制访问权限, 生成访问URL)       |
| - 文件导出服务 (PDF/Excel 生成)                   |
| - 认证/授权服务                                   |
+---------------------------------------------------+
           | Data Access Layer
           v
+-----------------------------+
|  用户数据库 (MySQL/PostgreSQL) |
|  - 报表元数据                 |
|  - 报表实例配置               |
|  - 数据源配置                 |
+-----------------------------+
```

**分层说明**:

1.  **前端 (Vue.js)**:
    *   提供用户操作界面，包括**报表设计器**（核心功能，用于可视化地定义报表的 SQL、图表/表格组件、布局等）、模板管理、报表创建配置、报表预览、报表发布管理、数据源管理等。
    *   **报表设计器**：用户在此界面通过拖拽、配置等方式定义报表的结构、SQL 查询、参数、可视化组件（图表、表格）、布局以及每个组件的展示属性。这些定义最终会生成一个 JSON 格式的报表配置元数据。
    *   **报表渲染器**：根据后端返回的报表配置元数据和数据，使用 ECharts、AntV G2/F2 等图表库和 Vue Table 组件进行渲染。
    *   通过 API 与后端服务进行交互。

2.  **后端 (Java)**:
    *   提供 RESTful API 供前端调用。
    *   **SQL 查询执行服务**: 接收前端（通过报表配置）传来的 SQL 查询和参数，**安全地执行查询**。这是核心服务之一，需要严格的 SQL 语句验证（只允许 SELECT）、权限隔离、查询超时控制等，确保数据安全和系统稳定。查询结果以 JSON 格式返回原始数据。
    *   **报表模板/实例管理服务**: 负责报表模板（即报表定义元数据）和报表实例（基于模板的具体配置）的 CRUD 操作。
    *   **数据源管理服务**: 统一管理各类数据库连接配置，敏感信息（如数据库密码）进行加密存储和访问控制。
    *   **报表发布服务**: 管理报表的发布状态、可见性和访问权限。
    *   **文件导出服务**: 负责将报表数据导出为 PDF 或 Excel 文件。
    *   与认证/授权系统集成，进行权限控制。

3.  **数据库**:
    *   存储报表模板的元数据（名称、描述、SQL 查询、参数定义、可视化配置、布局配置等）。
    *   存储报表实例的配置（基于哪个模板、数据过滤条件、参数值、发布状态、访问权限等）。
    *   存储数据源配置信息（连接字符串、用户名、密码等，敏感信息需加密）。

### 核心功能模块设计

#### 1. 自定义报表模板管理 (报表定义元数据管理)

*   **目标**: 允许用户定义报表的 SQL 查询、参数、可视化组件及其配置，并管理这些报表定义的元数据。
*   **实现方案**:
    *   **前端 (Vue)**:
        *   **报表设计器**: 提供一个可视化界面，用户可以：
            *   输入或构建 SQL 查询语句，并定义查询参数（例如 `${param1}`）。
            *   选择数据源。
            *   添加图表（柱状图、折线图、饼图、散点图等）、表格等组件。
            *   配置每个组件的数据映射（例如：图表的 X 轴映射到 SQL 结果的 `date` 字段，Y 轴映射到 `value` 字段；表格的列头和数据字段映射）。
            *   拖拽调整组件布局和大小。
            *   实时预览报表效果（通过调用后端 SQL 执行服务）。
        *   模板列表页面，展示所有报表定义及其元数据。
    *   **后端 (Java)**:
        *   **API**:
            *   `POST /api/report/templates`: 保存新的报表定义（包含 SQL、参数、可视化和布局配置 JSON）。
            *   `GET /api/report/templates`: 获取报表定义列表。
            *   `GET /api/report/templates/{id}`: 获取单个报表定义详情。
            *   `PUT /api/report/templates/{id}`: 更新报表定义。
            *   `DELETE /api/report/templates/{id}`: 删除报表定义。
        *   **数据模型**: `ReportDefinition` (id, name, description, category, sql_query, parameters_json, visualization_config_json, layout_config_json, create_time, update_time, status)。
            *   `parameters_json`: 存储 SQL 查询所需的参数定义 (e.g., `[{"name": "startDate", "type": "date", "defaultValue": "2023-01-01"}]`)。
            *   `visualization_config_json`: 存储图表/表格组件的配置，包括类型、数据映射、样式等。
            *   `layout_config_json`: 存储组件在页面上的布局信息。

#### 2. 报表创建管理 (报表实例配置管理)

*   **目标**: 基于已有的报表定义，创建具体的报表实例，并配置数据源、参数的实际值、过滤条件等。
*   **实现方案**:
    *   **前端 (Vue)**:
        *   提供报表创建向导，用户选择报表定义后，根据定义动态生成参数输入界面。
        *   允许用户选择数据源（从已配置的数据源中选择）。
        *   输入报表参数的实际值。
        *   报表预览功能（调用后端 SQL 执行服务获取数据，前端渲染）。
    *   **后端 (Java)**:
        *   **API**:
            *   `POST /api/reports`: 基于报表定义创建新的报表实例，保存配置信息到数据库。
            *   `GET /api/reports`: 获取报表实例列表。
            *   `GET /api/reports/{id}`: 获取单个报表实例详情及其配置。
            *   `PUT /api/reports/{id}`: 更新报表实例配置。
            *   `DELETE /api/reports/{id}`: 删除报表实例。
        *   **数据模型**: `ReportInstance` (id, definition_id, name, description, data_source_id, actual_parameters_json, create_time, update_time, status)。
            *   `actual_parameters_json`: 存储报表实例运行时传入的参数实际值。
        *   **SQL 查询执行服务**:
            *   接收 `reportInstanceId`，根据其 `definition_id` 获取 `sql_query` 和 `parameters_json`。
            *   接收 `actual_parameters_json`，将实际参数值替换到 `sql_query` 中的占位符。
            *   从 `data_source_id` 获取连接信息，执行 SQL，返回数据。
            *   **安全**: 严格限制 SQL 语句只能是 SELECT，并防止 SQL 注入。可以考虑白名单机制或 AST 解析。

#### 3. 报表发布管理

*   **目标**: 控制报表实例的可见性、访问权限，支持上下线操作，并提供发布后的报表访问入口。
*   **实现方案**:
    *   **前端 (Vue)**:
        *   提供报表发布管理界面，显示报表实例的发布状态、访问权限。
        *   提供发布/下线操作按钮。
        *   配置报表的访问用户或角色。
        *   生成报表访问 URL。
    *   **后端 (Java)**:
        *   **API**:
            *   `POST /api/reports/{id}/publish`: 发布报表，更新报表实例状态，并可能生成公开访问链接。
            *   `POST /api/reports/{id}/unpublish`: 下线报表。
            *   `PUT /api/reports/{id}/permissions`: 配置报表访问权限 (用户/角色)。
            *   `GET /api/reports/{id}/access-url`: 获取报表的访问 URL。
        *   **数据模型**: `ReportInstance` 增加 `publish_status`, `access_url`, `visibility` (public/private)。另需 `ReportPermission` 表 (report_id, user_id, role_id) 来管理权限。
        *   **认证/授权**: 平台已有的认证授权机制将用于控制用户对报表中心的访问以及对具体报表的访问权限。后端在处理报表访问请求时，需校验用户权限。
        *   **报表访问路由**: 当通过 `/reports/{reportId}` 访问报表时，后端会根据 `reportId` 获取报表定义和实例配置，然后获取数据，并将数据和渲染配置一并返回给前端页面进行渲染。

#### 4. 文件导出功能 (后端处理)

*   **目标**: 支持将报表数据或渲染后的报表导出为 PDF 和 Excel 文件。
*   **实现方案**:
    *   **前端 (Vue)**:
        *   提供导出按钮，选择导出格式 (PDF/Excel)。
        *   调用后端导出 API。
    *   **后端 (Java)**:
        *   **API**: `GET /api/reports/{id}/export?format={pdf|excel}`。
        *   **SQL 查询**: 根据 `reportId` 获取报表定义和实例配置，执行 SQL 查询获取原始数据。
        *   **PDF 导出**:
            *   使用 **iText** (或 Apache PDFBox)。
            *   或者，如果报表布局复杂，可以考虑将前端渲染的报表页面 (HTML) 通过 Headless Chrome (如 Selenium/Puppeteer) 转换为 PDF。
        *   **Excel 导出**:
            *   使用 **Apache POI** 库。根据报表配置中的列定义将数据写入 Excel。

### 技术选型及理由

*   **后端 (Java)**:
    *   **框架**: **Spring Boot / Spring Cloud**。提供快速开发、微服务架构支持，易于集成各类组件。
    *   **数据访问**: **MyBatis / JPA**。根据团队熟悉度选择，用于与数据库交互。
    *   **数据库**: **MySQL / PostgreSQL**。成熟稳定，功能强大，适合存储报表元数据和配置。
    *   **缓存**: **Redis**。用于缓存报表配置、SQL 查询结果（短时），提升性能。
    *   **文件导出**: **Apache POI** (Excel), **iText** (PDF)。若 PDF 复杂，考虑 **Headless Chrome + Puppeteer**。
    *   **安全**: Spring Security 或现有认证授权方案，确保 API 安全，特别是 SQL 查询执行服务的安全。
    *   **日志**: SLF4J + Logback，结合 ELK (Elasticsearch, Logstash, Kibana) 进行集中日志管理。

*   **前端 (Vue.js)**:
    *   **框架**: **Vue.js** (3.x 推荐)。用户已有经验，生态成熟。
    *   **UI 组件库**: **Element UI / Ant Design Vue**。提供丰富的组件，加速开发。
    *   **数据可视化库**: **ECharts / AntV G2 / Apache ECharts for Vue**。用于渲染图表。
    *   **表格组件**: **Element UI Table / Ant Design Vue Table / ag-Grid**。用于渲染数据表格。
    *   **报表设计器组件**: 这将是最大的挑战。可能需要集成一些开源的拖拽布局库（如 Vue Grid Layout）和表单配置生成器。

### 与现有系统的集成

1.  **认证与授权**:
    *   利用现有平台的统一认证和授权系统。
    *   通过 SSO (Single Sign-On) 或 token 传递，确保用户在平台和报表中心中的身份一致。
    *   后端报表服务在处理请求时，通过拦截器或过滤器校验用户权限。
    *   SQL 查询执行服务需要限制用户只能访问其有权限的数据源，甚至限制 SQL 操作类型（仅 SELECT）。

2.  **数据源管理**:
    *   平台可提供统一的数据源配置管理界面，由后端服务存储加密后的连接信息。
    *   在创建报表时，用户可选择平台已配置的数据源，后端服务将根据报表定义，使用对应的数据源连接执行 SQL。

3.  **报表访问**:
    *   发布的报表通过统一的访问 URL 提供，例如 `your-platform.com/reports/{reportId}`。
    *   后端服务在接收到此 URL 请求时，根据 `reportId` 获取报表定义和实例配置，执行 SQL 获取数据，并将渲染所需的 `visualization_config_json`, `layout_config_json` 和 `data` 一并返回给前端页面，由前端渲染器进行展示。

### 部署与运维考虑

*   **部署**: 报表中心后端服务、前端应用可以独立部署，通过容器化 (Docker, Kubernetes) 进行管理。
*   **监控**: 对报表服务的性能、SQL 查询执行时间、错误率、响应时间进行监控，确保服务的稳定性和报表生成效率。
*   **高可用**: 后端服务可通过多实例部署实现高可用。
*   **SQL 审计**: 对用户执行的 SQL 查询进行审计，记录查询内容、执行者、执行时间等。

### 后续扩展

*   **定时任务**: 报表定时生成、定时发送邮件等。
*   **报表订阅**: 用户可以订阅感兴趣的报表，定时接收更新。
*   **数据钻取/联动**: 报表内部或报表之间的数据联动和多层钻取功能。
*   **更强大的 SQL 编辑器**: 提供 SQL 语法高亮、自动补全、格式化等功能。
*   **数据模型层**: 在 SQL 裸奔之上，可以引入一个轻量级的数据模型层，让用户通过选择表、字段、连接关系来构建查询，降低直接写 SQL 的门槛。

### 验证方案

1.  **单元测试与集成测试**:
    *   后端服务对每个 API 接口和业务逻辑进行单元测试。
    *   对 SQL 查询执行服务进行重点测试，包括 SQL 注入防护、权限校验、查询超时等。
    *   对后端与数据库的集成、文件导出功能进行集成测试。
2.  **功能测试**:
    *   测试报表设计器：SQL 定义、参数定义、组件配置、布局调整、实时预览。
    *   测试报表实例的创建、参数配置、数据源选择、预览。
    *   测试报表的发布、下线、权限配置、访问。
    *   测试 PDF/Excel 导出功能。
3.  **性能测试**:
    *   测试大量数据、复杂 SQL 查询、高并发场景下报表生成和预览的性能。
    *   测试导出大文件时的性能。
4.  **安全测试**:
    *   测试数据源连接信息是否安全加密。
    *   测试 SQL 查询执行服务的安全性，防止恶意 SQL。
    *   测试权限控制是否严密，防止越权访问。
5.  **用户验收测试 (UAT)**:
    *   让最终用户实际使用报表中心，特别是报表设计器，收集反馈并进行优化。
